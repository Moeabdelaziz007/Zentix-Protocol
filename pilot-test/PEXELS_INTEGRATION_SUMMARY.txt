What was implemented:
-------------------
1. Real Pexels API integration in the searchImages function
2. Fallback mechanism to simulated images when API key is missing
3. Error handling for API failures
4. Environment configuration support
5. Informative logging

Key features:
-------------
- Uses real Pexels API when PEXELS_API_KEY is configured
- Falls back to placeholder images when API key is missing
- Handles API errors gracefully with fallback to simulation
- Supports configuration via .env file
- Maintains backward compatibility

How it works:
-------------
1. Server checks for PEXELS_API_KEY environment variable
2. If present, makes real API calls to Pexels
3. If absent or on error, uses simulated images
4. Returns array of image objects with URL and alt text

API Endpoint used:
-----------------
GET https://api.pexels.com/v1/search
Parameters:
- query: The search topic
- per_page: Number of images to return (default: 3)
- size: Image size (medium)

Response format:
----------------
[
  {
    "url": "https://images.pexels.com/photos/123456/pexels-photo-123456.jpeg",
    "alt": "Nature landscape"
  },
  ...
]

Environment setup:
------------------
1. Get API key from https://www.pexels.com/api/
2. Add to .env file:
   PEXELS_API_KEY=your_actual_api_key_here
